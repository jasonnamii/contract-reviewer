# D8 — AI 계약 도메인 (신설, 2026.5)

> 트리거: AI 학습데이터·LLM API·생성형 AI·MLOps·AI 영향평가·AI 위탁개발

## 1. 적용 법령 (1차 출처)
- **AI 기본법** (인공지능산업 진흥·신뢰기반 조성법, 2026.1.22 시행) — [https://www.msit.go.kr](https://www.msit.go.kr)
- **저작권법** §35의5 (공정이용), §76의2 (실연자권)
- **개인정보보호법** §15·§28의8 (학습데이터·국외이전)
- **EU AI Act** (2024.8 발효, 2026.2 전면시행) — 한국 기업 EU 진출 시
- **저작권법 시행령** (2026.6 개정 예정 — AI 학습 보상금)

## 2. 계약 유형 5종
### D8-1. AI 학습데이터 라이선싱 계약
**핵심 조항**:
- 데이터 사용 범위 (학습용·평가용·재배포)
- 저작권 귀속 + 라이선스 모델 (FRAND·perpetual·subscription)
- **2026 공정이용 부정 추세** (서울중앙지법 2025가합30000) — 무단 학습 ✗, 라이선싱 강제
- 손해배상 한도·면책
- 결과물(모델·embedding)의 사용권

**Redflag**:
- 「모든 권리 영구 양도」 + 추가 보상 ✗ = 매도자 불공정
- 「2차 학습·파인튜닝 무제한」 = 매수자 무제한 확장

### D8-2. LLM API 이용약관
**핵심 조항**:
- 입력(prompt) / 출력(completion) 권리
- **사용량 기반 과금 + Rate Limit + SLA**
- 데이터 학습 활용 옵트아웃 (Opt-out)
- **개인정보 처리 동의** + DPA (Data Processing Agreement)
- 콘텐츠 모더레이션 책임 분배

**Redflag**:
- 「입력 데이터 학습 이용 무제한」 = 영업비밀 유출
- 「출력 책임 사용자 전가」 = 사용자 패소 시 통째 부담
- 「약관 일방 변경권」 = 약관규제법 §6 위반 가능

### D8-3. AI 위탁개발 (SI·MLOps)
**핵심 조항**:
- **IP 귀속** (개발자 vs 발주자) — 디폴트: 발주자
- 학습데이터·모델·코드 분리 귀속
- 인수검사·하자담보 (모델 성능 SLA)
- **AI 영향평가** 책임 (2026.1 AI 기본법, 2027 본격) — 누가 수행
- 재학습·운영비 정산

**Redflag**:
- 「베이스 모델 개발자 보유」 + 「커스터마이징 부분만 발주자」 ⚠️ — 합리적이지만 명확화 필수
- 「영향평가 책임 미명시」 → 사고 시 책임 공방

### D8-4. AI 영향평가 위탁계약
**핵심 조항**:
- 평가 범위·방법론 (NIST AI RMF·EU AI Act·KISA 가이드)
- 평가자 자격·독립성
- 결과 보고서 IP·기밀유지
- **고위험 AI 인증**과 연계 (2027 본격)

### D8-5. 생성형 AI 콘텐츠 사용계약
**핵심 조항**:
- **AI 생성 표시 의무** (2026.1 AI 기본법 §31)
- 저작권 귀속 (AI 단독 생성물 = 한국 저작권 부여 ✗)
- 인격권·퍼블리시티 침해 면책
- 학습 사용 동의 / 옵트아웃

## 4. 표준 카운터 패턴
- **KAJ 패턴**: 「입력 데이터 학습 이용 무제한」 → 「(i) 영업비밀 표시 시 학습 제외, (ii) 사용자 옵트아웃 가능, (iii) 학습 사용 시 분기 보고」
- **태평양 패턴**: 「출력 책임 사용자」 → 「출력의 위법성·침해성 입증된 경우 한정, 양 당사자 과실 비율 적용」
- **광장 패턴**: 「IP 모두 개발자 귀속」 → 「베이스 모델 = 개발자 / 커스텀 부분 = 발주자 / 학습데이터 = 제공자」
