# M3 — 레드라인 (수정제안)

입력: 계약서 텍스트 + 입장(갑/을) + (옵션) Playbook.
출력: 조항별 Before→After + 코멘트 + 우선순위 + 통합 Summary.

## 흐름
① 계약서 전문 파싱
② Redflag 스크린 → 우선순위 설정
③ 각 수정 필요 조항: Before → After + 근거 코멘트
④ 추가 삽입 필요 조항 (Missing)
⑤ Summary·Negotiation Brief

## 산출 템플릿

```markdown

**작성일:** YYYY-MM-DD
**입장:** 갑/을
**원문 버전:** [v1 / 상대방 초안 / date]

---

## 1. 전체 Summary

### 1.1 수정 조항 수

- 수정 제안: M개

- **P1 (필수):** Red 조항 수정. 이거 관철 못하면 결렬.
- **P2 (중요):** Yellow 조항 개선
- **P3 (선택):** 표준화·명료화

---

## 2. 조항별 레드라인

### §X.Y [조항명] [P1/P2/P3]

- Redflag 분류: [예: Uncapped Liability (Red)]
- 위반·리스크: [예: 약관법 제8조 고객에게 불리한 손해배상 책임 면제]
```

## Redline 표기 규칙
- **삭제:** ~~취소선~~
- **추가:** **볼드** 또는 [추가]
- **교체:** ~~원문~~ **대체안**
- **주석:** `// 코멘트` 또는 `[Note: ...]`

## 우선순위 결정 기준
| 우선순위 | 기준 |
|--------|-----|
| P1 필수 | Redflag Red / 법령 위반 / 금액 큰 리스크 |
| P2 중요 | Redflag Yellow / 업계 표준 편차 / 실무 불편 |
| P3 선택 | 문구 명료화 / 용어 정의 / 스타일 |
